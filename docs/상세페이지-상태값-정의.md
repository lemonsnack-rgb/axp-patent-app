# 특허 상세페이지 상태값 정의서 (개발 전달용)

> 목적: 상세페이지의 **상태값·구분값(레이블값)**이 각 항목별로 **어떤 값이, 어떤 조건에서, 어떻게 표시되어야 하는지**를 개발자에게 정의한다.
> 성격: 완벽한 구현물이 아니라 **표시 규칙 정의**다. "가능한 값(enum)"·"표시 조건"·"실제 데이터 소스(수집 컬럼)"·"권장 표시형식"을 항목별로 명시한다. (현재 프로토타입 렌더: [src/components/PatentDetail.tsx](../src/components/PatentDetail.tsx), 목업 로직: [src/data/patentSeed.ts](../src/data/patentSeed.ts))

---

## 1. 핵심 — 문헌 상태(`status`)에서 파생되는 값 매트릭스

상세페이지 상태값의 다수는 **문헌 상태 `status`(등록/심사중/공개/거절/소멸)** 하나에서 결정된다. 개발 시 아래 대응을 그대로 정의로 사용한다.

| 문헌상태 `status` | 상태 Badge 색 | 권리상태 `rightStatus` | 최종처분상태 `finalDisposal` | 문헌종류 `docKind` | 등록번호·등록일 | 권리발생(`isReg`) |
|---|---|---|---|---|---|---|
| **등록** | 🟢 green | 존속 중 | 설정등록 | 등록특허공보 | 표시 | Y |
| **심사중** | 🟡 amber | 심사 중 | 심사청구 | 공개특허공보 | — | N |
| **공개** | 🔵 brand ⚠️ | 공개 | 출원공개 | 공개특허공보 | — | N |
| **거절** | ⚪ neutral | 거절확정 | 거절결정 | 공개특허공보 | — | N |
| **소멸** | ⚪ neutral | 소멸 | 존속기간만료 | 등록특허공보 | 표시 | Y |

- `isReg = (status === 등록 || status === 소멸)` — 등록번호·등록일·권리변동/이전·실시권 등 **권리 관련 항목의 표시 여부**를 결정한다.
- ⚠️ **불일치 발견**: `공개` 상태 Badge가 **상세페이지는 `brand`(파랑)**, **검색결과 목록은 `neutral`(회색)**로 서로 다르다.
  - 상세: [PatentDetail.tsx:48-51](../src/components/PatentDetail.tsx#L48-L51) → 공개=`brand`
  - 공용 유틸: [badgeUtils.ts `getPatentStatusBadgeColor`](../src/utils/badgeUtils.ts) → 공개=`neutral`
  - **정의(권장): 공통 유틸 `getPatentStatusBadgeColor`로 일원화**하여 전 화면 동일 색을 쓴다. (공개=neutral 권장, 또는 brand로 통일 — 택1)

### 상태값 툴팁(의미) — 정의됨
상태 Badge에는 hover 설명을 붙인다. 정의는 [badgeUtils.ts `PATENT_STATUS_DESC`](../src/utils/badgeUtils.ts):

| 상태 | 설명(툴팁) |
|---|---|
| 등록 | 특허권이 설정등록되어 현재 존속 중인 상태 |
| 심사중 | 출원 후 심사가 진행 중인 상태(권리 미발생) |
| 공개 | 출원이 공개된 상태(심사 전/중, 권리 미발생) |
| 거절 | 심사 결과 거절결정된 상태 |
| 소멸 | 존속기간 만료·포기·취소 등으로 권리가 소멸된 상태 |

> 권장: 권리상태·최종처분상태에도 동일한 방식의 짧은 의미 툴팁을 제공한다.

### 실제 데이터 소스 (수집 명세서 대응)
- `status`/`rightStatus` ← `bibliographic.register_status` + `custom.legal_status`
- `finalDisposal` ← `bibliographic.final_disposal`
- `docKind` ← `bibliographic.document_kind`
- 등록번호·등록일 ← `bibliographic.register_number / register_date`

## 2. 독립 상태값·구분값 (status 파생 아님)

| 항목 | 화면 위치 | 가능한 값(enum) | 표시 조건 | 실제 소스 | 권장 표시형식 |
|---|---|---|---|---|---|
| **국가 `country`** | 상단·패밀리·우선권 | KR / US / JP / CN / EP (WIPO 등) | 항상 | `bibliographic.country_code` | Badge `brand` |
| **권리변동 유무 `rightChange`** | 서지 | `있음 (권리 양도)` / `없음` | 항상 | `custom.has_ownership_change`(Y/N) | 있음=Badge `amber`, 없음=평문 `—` 권장 |
| **심판 `trial`** | 기타 > 심판 | `심판 없음` / `무효심판 계속 중 (YYYY당NNNN)` | 항상(없음이면 섹션 숨김) | `custom.has_trial`(Y/N) → 유형 | 있음=Badge `amber` |
| **권리변동 이력 구분 `type`** | 기타 > 권리변동 이력 | 권리 양도 (외 권리소멸·명의변경 등) | `isReg`일 때 | `RIGHT_CHANGE.change_type` | Badge `neutral` |
| **행정처리 상태 `proc_status`** | 기타 > 행정처리 | 수리 / 발송 (외 수신·완료 등) | 항상(출원서), status별 추가 | `ADMINISTRATIVE_PROCESS.proc_status` | Badge `neutral` |
| **US 관련출원 상태 `status`** | 기타 > 국가별(US) | Granted (외 Pending·Abandoned 등) | `country===US` | `REL_APPL.status` | Badge `neutral` |

### 행정처리 상태의 표시 조건 (status 연동)
| 문서 | 표시 조건 | 상태값 |
|---|---|---|
| 출원서 | 항상 | 수리 |
| 의견제출통지서 | `status !== 공개` | 발송 |
| 등록결정서 | `isReg`(등록/소멸) | 발송 |

## 3. 값이 아니라 유무로 갈리는 항목 (표시/숨김 정의)

아래 항목은 "상태값"보다 **있음/없음(섹션 노출 여부)**이 핵심이다. 조건은 [목업-조건표.md](목업-조건표.md)와 동일.

| 항목 | 노출 조건 | 소스 |
|---|---|---|
| 우선권 주장 | country ∈ {US, EP} (또는 priority 보유) | `priority` / `PRIR` |
| 지정국 | country === EP | `DESIGNATED_COUNTRY` / `DSGN` |
| 서열목록 | `custom.sequence_listing_yn === Y` | `custom.sequence_listing_yn` |
| 국가 R&D | KR & 과제 보유 | `RND` |
| 표준특허 | 표준 선언 보유 | `STANDARD` |
| 실시권 등록일 | `isReg` & 실시권 보유 | `custom.license_registration_date` |
| 선행기술문헌 | 인용문헌 보유 | `PRIOR_TECHNOLOGY_DOCUMENT` |

## 4. 표시형식 일관성 — 현재 상태 및 정의

현재 상태값 표시가 **Badge / 평문 혼재**다. 개발 정의로 아래 규칙을 채택한다.

- **Badge로 표시(상태·구분값):** 문헌상태, 국가, 권리상태, 최종처분상태, 심판(있음), 권리변동 구분, 행정처리 상태, US 관련출원 상태
- **평문으로 표시(단순 스칼라):** 문헌종류, 청구항 수, 도면 수, 각종 번호/일자
- **색 규칙:** 문헌상태=의미색(green/amber/neutral) · 국가=brand · 그 외 구분값=neutral · 경고성(심판 있음·권리변동 있음)=amber
- **불일치 조치:** 1절의 `공개` 상태 색을 공통 유틸로 일원화.

> 현재 프로토타입은 권리상태·최종처분상태·문헌종류를 **평문**으로 표시 중이다. 위 정의(권리상태·최종처분상태 → Badge)로 맞추려면 코드 수정이 필요하며, 별도 요청 시 적용한다.
