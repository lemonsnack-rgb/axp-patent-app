# 특허 상세페이지 목업 조건표

> ⚠️ **이 문서의 성격 — 개발 전달용 기능 설명서 (구현 명세 아님)**
>
> 본 문서는 프로토타입이 **어떤 필드를 어떤 상황에서 화면에 보여줘야 하는지**를 개발자에게 설명하기 위한 것이다. 기능의 완벽한 구현물이 아니다.
> - 아래의 채움 조건(`국가(slot)`·`상태(isReg)`·`seq % N`)은 **데모에서 모든 필드가 최소 1건씩 화면에 나타나도록** 인위적으로 만든 시드 규칙이다. **실제 서비스의 비즈니스 로직이 아니다.**
> - 실제 구현에서는 이 값들이 **수집 DB의 해당 컬럼**에서 채워진다. "EP만 지정국", "US만 관련출원"처럼 **국가/문헌 특성에 따라 있고 없음이 갈리는 관계**만 참고하고, `seq % 6` 같은 규칙은 데모 재현용으로만 읽는다.
>
> 대상: `[BLK] 특허 수집 데이터베이스 명세서` 대조 결과 상세페이지에 보강한 필드 중 **조건부로만 채워지는 목업 필드**. (데모 소스: [src/data/patentSeed.ts](../src/data/patentSeed.ts) `buildPatent`)

## 1. 데이터 구조 (재현의 기준)

시드는 **10개 기술 도메인 × 5건 = 50건**을 결정론적으로 생성한다. 랜덤/날짜 함수 없이 `seq = domIdx*5 + slot` 하나로 모든 값이 정해진다.

| 변수 | 규칙 |
|---|---|
| `slot` | 각 도메인 내 0~4번째 |
| `국가(cc)` | **slot 고정** → `0=KR · 1=US · 2=JP · 3=CN · 4=EP` |
| `status` | `STATUS_SEQ[(domIdx+slot)%5]` → `0=등록 · 1=심사중 · 2=공개 · 3=거절 · 4=소멸` |
| `isReg` | status가 **등록 또는 소멸**일 때 true (권리 발생) |
| `seq` | `domIdx*5 + slot` (0~49) |

### 도메인 순서(domIdx)
`0 라이다 · 1 이차전지 · 2 반도체메모리 · 3 그래핀복합소재 · 4 5G빔포밍 · 5 의료영상 · 6 마이크로LED · 7 수전해 · 8 협동로봇 · 9 트랜스포머영상`

### 검증의 핵심 규칙
- **국가는 위치로 고정된다.** 어떤 도메인이든 검색 결과 5건의 순서는 항상 `KR → US → JP → CN → EP`. 따라서 특정 국가 필드를 보려면 그 위치의 문헌을 열면 된다.
- **"라이다" 도메인 1개만 열면 5개 status를 모두 커버한다.**
  - 1번째 KR = **등록**, 2번째 US = **심사중**, 3번째 JP = **공개**, 4번째 CN = **거절**, 5번째 EP = **소멸**

## 2. 필드별 조건표

### 2-1. 서지사항

| 필드 | 채워지는 조건 | 예시로 여는 방법 |
|---|---|---|
| 최종처분상태 `finalDisposal` | 항상(status별 값) | 등록=설정등록 / 거절=거절결정 / 소멸=존속기간만료 / 심사중=심사청구 / 공개=출원공개 → 라이다 5건 |
| 청구항 수 `claimCount` | 항상 | 모든 문헌 |
| 도면 수 `drawingCount` | 항상(도면 개수) | 모든 문헌 |
| **출원구분 `applicationFlag`** | EP=`PCT 국내단계진입`, CN(slot3)=`분할출원`, 그 외=`정상출원` | PCT→라이다 5번째(EP) / 분할→라이다 4번째(CN) / 정상→라이다 1번째(KR) |
| **원출원번호 `originalAppNo`** | **CN(slot3)에만** 값, 그 외 `-` | 라이다 4번째(CN) |
| 국제출원번호·국제출원일 `intlAppNo/intlAppDate` | **EP만** | 라이다 5번째(EP) |
| **번역문 제출일 `translationSubmitDate`** | **KR 외 전 국가**(US/JP/CN/EP) | 있음→라이다 2번째(US) / 없음→라이다 1번째(KR) |
| 특허고객번호 `customerNo` | **KR만**, 그 외 `-` | 라이다 1번째(KR) |
| **지정국 `designatedCountries`** | **EP만**(DE/FR/GB/IT/NL) | 라이다 5번째(EP) |
| **실시권 등록일 `licenseRegDate`** | `isReg && seq 짝수` | 있음→라이다 1번째(KR, seq0) / 없음→홀수 seq 등록문헌 |
| **서열목록 `sequenceListing`** | `seq % 7 === 0`(바이오/서열) | 라이다 1번째(seq0), 이차전지 3번째(seq7, JP) |

### 2-2. 분류코드

| 필드 | 채워지는 조건 | 예시로 여는 방법 |
|---|---|---|
| IPC / CPC | 항상(CPC는 JP 제외) | 모든 문헌 (JP는 CPC `-`) |
| **국가 고유 분류 `countryClassifications`** | JP=`FI·F-term·테마`, US=`UPC`, EP=`EPC`, KR/CN=없음 | JP→라이다 3번째 / US→라이다 2번째 / EP→라이다 5번째 |

### 2-3. 기타정보

| 필드 | 채워지는 조건 | 예시로 여는 방법 |
|---|---|---|
| **권리변동 이력 `rightChangeList`** | `isReg`(등록/소멸) | 라이다 1번째(KR등록), 5번째(EP소멸) |
| **권리이전 이력 `rightTransferList`** | `isReg`(등록/소멸) | 라이다 1번째(KR등록), 5번째(EP소멸) |
| 행정처리(수발신) `adminProcess` | 출원서(항상) + 의견제출통지서(status≠공개) + 등록결정서(isReg) | 3개→라이다 1번째(등록) / 출원서만→라이다 3번째(JP,공개) |
| 국가 R&D `rnd` | **KR & seq 짝수** | 라이다 1번째(KR seq0). 이차전지 1번째(KR seq5)는 홀수→없음 |
| 표준특허 `standard` | `seq % 6 === 0` | 라이다 1번째(seq0), 이차전지 2번째(seq6, US) |
| 심판/소송 | 심판=`seq%9===0`, 분쟁=`seq%7===0` | 분쟁→라이다 1번째(seq0) |
| 특허평가 등급 `grade` | 항상 | 모든 문헌 |

### 2-4. 국가별 추가정보 (기타정보 하위)

| 필드 | 채워지는 조건 | 예시로 여는 방법 |
|---|---|---|
| **US 관련출원 `usRelatedApps`** | **US만** | 라이다 2번째(US) |
| **US 가출원 `usProvisional`** | **US만** | 라이다 2번째(US) |
| **JP 공보판 `jpEdition`** | **JP만**(등록=특허공보(B) / 그외=공개특허공보(A)) | 공개(A)→라이다 3번째(JP) / 등록(B)→그래핀 3번째(JP, 등록) |
| **JP 대리인 구분 `agentCategory`** | **JP만**(弁理士) | 라이다 3번째(JP) |
| **EP 출원인 정리번호 `epFileRef`** | **EP만** | 라이다 5번째(EP) |
| **EP 출원/공개 언어 `epFilingLanguage`** | **EP만**(seq%3: 0=de·1=en·2=fr) | en→라이다 5번째(seq4) / de→이차전지 5번째(seq9) |

### 2-5. 인용·피인용

| 필드 | 채워지는 조건 | 예시로 여는 방법 |
|---|---|---|

## 3. "전부 채워진" 확인용 대표 문헌

- **라이다 1번째 (KR / 등록 / seq0)** — 조건부 필드가 가장 많이 켜지는 문헌:
  특허고객번호(KR)·국가R&D(KR&짝수)·표준특허(seq%6)·서열목록(seq%7)·실시권등록일(isReg&짝수)·권리변동·권리이전·분쟁·행정처리 3종.
- **라이다 5번째 (EP / 소멸 / seq4)** — EP 고유 필드 총집합:
  국제출원번호·지정국·EPC 분류·EP 정리번호·EP 언어·우선권(EP)·번역문제출일·권리변동/이전(isReg).
- **라이다 2번째 (US / 심사중 / seq1)** — US 고유:
  US 관련출원·US 가출원·UPC 분류·우선권(US)·번역문제출일.
- **라이다 3번째 (JP / 공개 / seq2)** — JP 고유:
  JP 공보판(A)·대리인구분·FI/F-term/테마 분류. (공개라 의견제출통지서 없음 → 행정처리는 출원서만)
